## 1. 제품 개요

| 항목 | 내용 |
|---|---|
| **제품명** | 워터링 (Watering) |
| **플랫폼** | Android 13+ (API 33+) |
| **카테고리** | 건강 & 피트니스 |
| **타겟 시장** | 한국 우선, 글로벌 확장 |
| **핵심 가치** | 잠금화면을 볼 때마다 자연스럽게 물 마시기를 상기시킨다 |
| **포지셔닝** | Android 최초 잠금화면 위젯 퍼스트 물 마시기 앱 |

### 1-1. 제품 비전

> 스마트폰 잠금화면은 하루 평균 80번 이상 본다.
> 워터링은 이 순간을 활용해 별도의 노력 없이 수분 섭취 습관을 만든다.
>
> Android 16 QPR1부터 잠금화면 위젯이 부활했다.
> 경쟁앱 대부분이 이 변화를 따라오지 못한 지금이 선점 기회다.

### 1-2. 핵심 지표 (North Star Metric)

**7일 연속 달성 유저 비율** — iOS 버전과 동일. 리텐션, 구독 전환, 입소문 모두 이 지표에서 나온다.

## 2. iOS→Android 전환 배경

### 2-1. 전환 이유

| 이유 | 설명 |
|---|---|
| **Apple Developer 블로커** | 개인 계정 등록 거절 (케이스 102924028092), 원인 불명 |
| **Android 진입 장벽 낮음** | Google Play 등록비 $25 일회성 (vs Apple $99/년) |
| **잠금화면 위젯 선점 기회** | Android 16 QPR1(2025년 9월)로 잠금화면 위젯 부활, 경쟁앱 미적용 상태 |
| **Android 시장 점유율** | 한국 Android 점유율 약 75%, 글로벌 72% |

### 2-2. iOS 대비 Android의 기회

- 잠금화면 위젯이 Android에서 **12년만에 복귀** (Android 4.2 이후 제거됐다 Android 16 QPR1에서 부활)
- 기존 경쟁앱(Hydro Coach, Water Drink Reminder 등) 대부분 **잠금화면 위젯 미지원**
- 한국 시장에서 **Android 유저가 더 많음**

### 3-3. 경쟁앱 공통 약점 (기회 영역)

1. **잠금화면 위젯 없음** — 모두 홈화면 위젯만 지원
2. **한국 특화 없음** — 미세먼지/폭염 연동 없음
3. **1탭 기록 불가** — 위젯에서 바로 기록 안 됨
4. **가격 불투명** — 갑작스러운 가격 인상 (Hydro Coach 부정 리뷰 다수)
5. **개인정보 우려** — 클라우드 동기화 강제

### 3-4. Android 잠금화면 위젯 현황

- **Android 16 QPR1** (2025년 9월 배포): 스마트폰용 잠금화면 위젯 공식 지원
- 기존 홈화면 위젯과 동일 API 기반, 기존 위젯은 잠금화면 자동 호환
- 동적 리사이징, 컬러링, 기존 위젯 타입 전부 호환
- **현재 시점 기준 경쟁앱 중 잠금화면 위젯 전면 지원 앱 없음**

## 4. 사용자 페르소나

iOS 버전과 동일한 3개 페르소나를 유지하되, Android 특성 반영

### 페르소나 A — 주 타겟: "바쁜 직장인 지은"

```
나이:     29세
직업:     스타트업 마케터
기기:     갤럭시 S24 (Android)
상황:     하루 종일 컴퓨터 앞에 앉아있고 물 마시는 걸 자주 잊음
Pain:     알림이 와도 귀찮아서 무시, 습관 앱을 깔아도 3일 만에 포기
Goal:     피부가 건조하고 오후에 집중력이 떨어지는 문제를 해결하고 싶음
행동 패턴: 잠금화면을 수시로 보지만 특별히 앱을 여는 행동은 잘 안 함
결제 성향: 월 5,000원 이하면 가치 있으면 결제
핵심 니즈: 귀찮지 않게, 의식하지 않아도 알아서 되는 것
```

### 페르소나 B — 서브 타겟: "건강 챙기는 30대 준혁"

```
나이:     34세
직업:     개발자 (재택근무)
기기:     픽셀 8 Pro (Android)
상황:     건강에 관심이 생겼고 운동도 시작, 수분 섭취도 체계적으로 관리하고 싶음
Pain:     기존 앱은 너무 복잡하거나 디자인이 구림
Goal:     데이터 기반으로 건강 루틴을 만들고 싶음 (Health Connect 연동 원함)
행동 패턴: 위젯을 적극 활용하고 앱 꾸미기에 관심 있음
결제 성향: 가치 있으면 연간권도 결제함
핵심 니즈: 깔끔한 디자인 + 데이터 통계
```

### 페르소나 C — 잠재 타겟: "다이어트 중인 수연"

```
나이:     26세
직업:     대학원생
기기:     갤럭시 A55 (Android)
상황:     다이어트 중 식욕 억제를 위해 물을 의식적으로 마시려 함
Pain:     경쟁 앱이 체중 감량 포커스라 불편함 (다이어트 tips 강요)
Goal:     부담 없이 수분 섭취량만 관리하고 싶음
행동 패턴: SNS에서 앱 발견, 예쁜 위젯 스크린샷에 반응
결제 성향: 무료로 쓰다가 예쁜 테마 나오면 결제 고려
핵심 니즈: 다이어트 프레이밍 없이 깔끔하게
```

## 5. 핵심 기능 요구사항

> **개발 원칙**: 기본 기능에 충실하게, 직관적이고 심미적인 UI를 우선한다.
> 미세먼지/폭염 API, 위젯 테마 커스터마이징 등 복잡한 외부 의존 기능은 v2 이후로 연기.

### 5-1. 잠금화면 & 홈화면 위젯 (MVP 핵심)

Android의 위젯 시스템은 Glance API (Jetpack Glance)를 사용한다.

| 위젯 종류 | 크기 | 표시 정보 | iOS 대응 |
|---|---|---|---|
| **원형 위젯** | **1×1** | 물방울 아이콘 + 잔 수 + 목표 + 진행 바 | Circular |
| **직사각형 위젯** | **2×2** | 잔 수 / 목표 + % + 진행 바 + 동기부여 문구 | Rectangular |
| **좁은 위젯** | **2×1** | 아이콘 + 잔수 + 가로 진행 바 + % | Inline |

**위젯 디자인:**
- 다크 모드: 다크 배경(`#0D1B2A`) + 아쿠아(`#00B4D8`) / 그린(`#34C759`) 강조색
- 라이트 모드: 흰 배경 + 아쿠아 / 그린 텍스트 (OS 설정 자동 감지)
- 목표 달성 시 아쿠아 → 그린 자동 전환
- 달성률에 따른 색상 변화: 미달성 → 아쿠아, 목표 달성 → 그린
- 위젯 탭 → 딥링크(`watering://quick-record`) → 앱 진입 즉시 +1잔 자동 기록 + 스낵바
- 자정 기준 자동 초기화 (WorkManager 스케줄)
- **잠금화면 위젯**: Android 16 QPR1+ 기기에서 잠금화면에도 위젯 추가 가능
  - 홈화면 위젯과 동일 구현체 재사용 (Android 잠금화면 위젯은 홈화면 위젯과 동일 API)
  - Target SDK 36+ 및 `android:widgetCategory="keyguard|home_screen"` 선언 필요

### 5-2. 빠른 기록 UX

- **홈 화면 1탭 버튼**: "물 마셨어요 💧" → 설정된 컵 크기만큼 즉시 기록
- **위젯 탭**: 앱 열림 → 자동 기록 → 스낵바 2초 표시
- **음료 선택 시트**: 6종(물/커피/주스/차/우유/기타) + 양 6종(100~500ml) 선택 (ModalBottomSheet)
- **기록 취소**: 홈 화면에서 마지막 기록 즉시 취소 (Snackbar with Undo action)

### 5-3. 목표 설정

- 하루 목표 잔 수 (+/- 버튼: 1~20잔)
- 컵 크기 선택 (150/200/250/300/350/500ml)
- 한국 보건복지부 기준 기본값: 8잔 × 200ml = 1,600ml

### 5-4. 알림 시스템 (전체 무료)

- 알림 ON/OFF
- 시작 시간 / 종료 시간 설정
- 알림 간격 선택: 30분 / 1시간 / 1.5시간 / 2시간 / 3시간 / 4시간
- WorkManager로 정확한 알림 스케줄 구현
- Android 13+: POST_NOTIFICATIONS 권한 런타임 요청 필수
- 알림 권한 미승인 시 설정 화면 내 배너 안내

> **결정 이유**: 알림 간격 커스텀은 경쟁앱에서 유료 기능이나,
> 워터링은 "마찰 없는 무료 경험"을 차별화 포인트로 삼는다.

### 5-5. 히스토리 & 통계

**무료:**
- 오늘 기록 타임라인 (음료 종류, 시간, 양)
- 7일 달성률 바 차트 (Vico 또는 MPAndroidChart)
- 주간 요약: 평균 잔 수, 달성한 날 수, 총 섭취량, 최장 스트릭

**프리미엄 (v1 출시 후 구현):**
- 30일 달성률 트렌드
- 시간대별 패턴 분석 ("오후 2~4시에 자주 빠뜨려요")
- 요일별 달성률 히트맵
- 데이터 내보내기 (CSV)

### 5-6. Health Connect 연동 (무료)

iOS의 HealthKit에 대응하는 Android의 표준 건강 데이터 플랫폼.

- 물 마시기 기록 → Health Connect 자동 저장 (`HydrationRecord`)
- 권한 거부해도 앱 모든 기능 정상 작동 보장
- Android 14+: 시스템 내장 / Android 9~13: Google Play에서 별도 설치
- 무료 제공 → Google Play "건강" 카테고리 노출 강화

### 5-7. 스트릭 & 동기부여

**무료:**
- 오늘 현재 연속 달성일 / 최장 연속 달성일 표시
- 목표 달성 시 위젯 색상 초록 전환
- **성취 타이틀 시스템** — 특정 조건 달성 시 풀스크린 애니메이션 모달로 타이틀 수여

| 타이틀 | 이모지 | 조건 |
|---|---|---|
| 시작의 한 모금 | 🌱 | 오늘 첫 잔 기록 |
| 반환점 돌파! | 💧 | 목표 50% 달성 |
| 오늘 하루 완벽했어요 | 🏆 | 오늘 목표 100% 달성 |
| 3일 연속 달성! | 🔥 | 목표 달성 + 연속 3일 |
| 일주일 챔피언 | ⚡ | 목표 달성 + 연속 7일 |
| 한 달 레전드 | 👑 | 목표 달성 + 연속 30일 |

- 같은 타이틀은 하루에 한 번만 표시 (DataStore 중복 방지)
- 스트릭 타이틀은 일반 타이틀보다 우선 표시 (더 특별함)

**프리미엄:**
- 스트릭 보호권 (월 1회) — 하루 빠져도 연속 유지

## 6. UI/UX 설계

### 6-1. 디자인 원칙

> 워터링은 "가장 예쁜 잠금화면에서, 가장 쉽게 기록하는 물 마시기 앱"이다.
> Material Design 3 (Material You)을 기반으로 하되, 물이라는 소재의 블루 톤 아이덴티티를 유지한다.

| 원칙 | 설명 |
|---|---|
| **직관적** | 설명 없이도 사용법을 알 수 있어야 한다. 버튼 1개로 기록 완료. |
| **심미적** | Material Design 3 + Dynamic Color (Android 12+) 지원. 기기 배경화면 색상에 앱 테마 자동 적용. |
| **빠름** | 기록까지 1탭. 앱 열어서 찾는 시간 0초. |
| **조용함** | 불필요한 팝업, 광고, 강요 없음. 사용자를 방해하지 않는다. |
| **신뢰감** | 서버 없음, 계정 없음, 광고 없음. 내 데이터는 내 기기에만. |

### 6-2. Material Design 3 적용

iOS 대비 Android 특화 설계:

| iOS | Android |
|---|---|
| SF Pro 폰트 | Roboto / Noto Sans KR |
| SF Symbols | Material Symbols (outlined) |
| Navigation Stack | Navigation Component (Bottom Nav) |
| Sheet | ModalBottomSheet |
| Toast | Snackbar |
| `.spring()` 애니메이션 | Spring Animation (MotionLayout) |
| System 컬러 (Dynamic) | Dynamic Color (Material You, Android 12+) |

### 6-3. 사용자 플로우

```
첫 실행
  └─ 온보딩 Step 1: 앱 소개 + 핵심 기능 3가지
       └─ 온보딩 Step 2: 하루 목표 설정 (잔 수 / 컵 크기)
            └─ 온보딩 Step 3: 알림 권한 요청 (POST_NOTIFICATIONS)
                 └─ 온보딩 Step 4: Health Connect 연동 여부 선택
                      └─ 홈 화면

일반 사용 (잠금화면, Android 16 QPR1+)
  잠금화면 위젯으로 진행률 확인
    └─ 위젯 탭 → 앱 진입 → 자동 +1잔 기록 → 스낵바 표시
         └─ 위젯 즉시 갱신
```

### 7-4. DataStore 설계 (앱↔위젯 데이터 공유)

Android에서 앱과 위젯은 **같은 프로세스** 또는 **ContentProvider**로 데이터 공유.

```
저장 방식: Proto DataStore (타입 안전) 또는 Preferences DataStore
파일 위치: Context.filesDir (앱 내부 저장소)

공유 데이터 구조:
  - today_record    → DayRecord (오늘 기록, JSON 직렬화)
  - user_settings   → UserSettings
  - streak_info     → StreakInfo (현재/최장/보호권 여부)
  - record_history  → Map<String, DayRecord> (30일 히스토리)
  - last_updated    → Long (epoch millis, 위젯 갱신 기준)
  - is_onboarding_done → Boolean

위젯 갱신 방식:
  - 기록 저장 후 GlanceAppWidgetManager.updateAll() 호출
  - 자정 초기화: WorkManager 일일 작업 스케줄
```

```kotlin
// Glance 위젯 기본 구조
class WateringWidget : GlanceAppWidget() {

        provideContent {
            WateringWidgetContent()
        }
    }

suspend fun updateAllWidgets(context: Context) {

        .getGlanceIds(WateringWidget::class.java)
        .forEach { WateringWidget().update(context, it) }
}
```

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|---|---|---|---|
| 잠금화면 위젯 제조사별 지원 편차 | 높음 | 중간 | 삼성 One UI, 픽셀 우선 테스트, 미지원 기기에 안내 문구 |

| WorkManager 배터리 최적화 충돌 | 중간 | 높음 | 정확한 알림은 AlarmManager + SCHEDULE_EXACT_ALARM 권한 병행 |
| Glance API 위젯 갱신 지연 | 중간 | 중간 | 갱신 후 마지막 갱신 시간 표시 |
| Google Play Billing 검증 실패 | 낮음 | 높음 | 서버 없이 Google Play 실시간 구매 검증 API 활용 |
| Health Connect API 변경 | 낮음 | 중간 | Android 버전별 분기 처리, Google 베타 조기 테스트 |

### 12-2. 비즈니스 리스크

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|---|---|---|---|
| 경쟁앱의 잠금화면 위젯 빠른 추가 | 중간 | 높음 | 선점 이후 UX 완성도로 차별화 |
| Google Play 심사 거절 | 낮음 | 높음 | 가이드라인 사전 준수, Health Connect 권한 정책 준수 |
| 낮은 프리미엄 전환율 | 중간 | 높음 | 무료 기능 조정, 업셀 UX 개선 |
| Android 특유의 낮은 결제 전환율 | 중간 | 높음 | iOS 대비 Android 결제 전환율 낮음 → 무료 유저 수로 보완, 광고 없이 브랜드 신뢰 구축 |

### 12-3. Android 특화 정책 리스크

- **Health Connect 권한 정책**: 민감 권한으로 Google Play 특별 심사 대상. 앱 내 사용 목적 명시 필수.
- **SCHEDULE_EXACT_ALARM**: Android 12+에서 정확한 알람 권한 별도 요청 필요. 미부여 시 WorkManager fallback.
- **Battery Optimization**: 삼성 등 일부 제조사의 배터리 최적화가 WorkManager를 kill할 수 있음. 사용자에게 배터리 예외 설정 안내 필요.

## 13. 법적 & 정책 요건

### 13-1. 필수 문서

| 문서 | 내용 |
|---|---|
| 개인정보처리방침 | 수집 데이터 없음(로컬 only) 명시, Google Play URL 필수 |
| 이용약관 | 구독 조건, 환불 정책, 서비스 변경 고지 |
| 구독 취소 안내 | Google Play 구독 관리에서 취소 방법 인앱 표시 |

### 13-2. 개인정보처리방침 핵심 내용

```
수집하는 개인정보: 없음
서버 전송 데이터: 없음
제3자 제공: 없음
데이터 저장 위치: 사용자 기기 내부만
Health Connect 데이터: 기기 외부로 전송하지 않음
광고 식별자(GAID): 사용 안 함
```

> iOS용 Google Sites 개인정보처리방침 페이지를 Android도 포함하도록 내용 추가

### 13-3. Health Connect 사용 정책

- Google Play에서 Health Connect 사용 앱 별도 심사 (민감 권한)
- 권한 요청 목적을 앱 내 명확히 설명 (읽기/쓰기 각각)
- Health Connect 데이터를 광고나 제3자에게 절대 제공 금지
- 사용자가 권한 거부해도 앱 기본 기능 작동 보장

### 13-4. Google Play 구독 정책

- 무료 체험 기간 명확히 표시 (7일)
- 체험 종료 후 자동 결제 사전 고지
- 연간 구독 자동 갱신 안내 (Google Play 처리)
- 구독 관리 딥링크 인앱 제공
