## 지원 범위 (Android 버전 / 기기)

| 항목 | 값 |
|---|---|
| minSdk | 26 (Android 8.0 Oreo, 2017) |
| targetSdk / compileSdk | 36 (Android 16) |
| 실질 커버리지 | 활성 기기의 99%+ (2026년 기준 Android 8.0 미만은 시장에서 사실상 소멸) |
| 기기 제약 | `AndroidManifest.xml`에 `<uses-feature>`/`<supports-screens>` 제약 없음 — Android 8.0+ 폰/태블릿 전체 설치 가능 (태블릿 UI는 v1 미최적화) |

> **⚠️ 문서 불일치 발견**: `docs/기획서.md` 1장에는 "플랫폼: Android 13+ (API 33+)"로 기재되어 있으나 실제 `app/build.gradle.kts`는 `minSdk = 26`. 실빌드 기준이 기획서보다 더 넓은 범위를 지원 중이므로 유리한 방향의 불일치지만, 기획서 쪽 업데이트 필요.

**버전별 기능 제약**:
- `POST_NOTIFICATIONS` 런타임 권한 요청은 Android 13(API 33)+ 에서만 발생 (그 미만은 항상 허용 상태로 동작)
- 잠금화면 위젯(`widgetCategory="keyguard"` 선언)은 Android 16 QPR1 정식 출시 후에만 실제 동작 — 현재는 매니페스트 선언만 되어 있고 미검증 상태
- Health Connect 연동은 별도 Health Connect 앱 설치 필요 (P3, 미구현)

## 핵심 모듈 설명

### RecordSheet 플로우
```
OutlinedButton 클릭 (HomeScreen)
  → showRecordSheet = true
  → RecordSheet(ModalBottomSheet) 표시
    → RecordViewModel이 DrinkType / 용량 상태 관리
    → "기록하기" 버튼 → onRecord(amount, drinkType) 콜백
      → HomeViewModel.addWaterCustom(amount, drinkType)
        → WaterService.addWater() → DataStore 저장 → 위젯 갱신
      → onDismiss() → showRecordSheet = false
```

### DrinkType computed properties
`emoji`와 `displayName`은 `get()` 형태의 computed property로, `@Serializable`과 충돌 없이 안전하게 추가됨.

### Google Play Billing 구독 (BillingService + PremiumScreen)

- **BillingService** (`@Singleton`, `PurchasesUpdatedListener` 구현):
  - `startConnection()`: 앱 시작 시 `BillingClient` 연결 → 성공하면 `loadProducts()` + `queryPurchases()` 호출
  - `loadProducts()`: 구독 2종(SUBS) + 평생 상품 1종(INAPP) `queryProductDetails`
  - `launchPurchase()`: 구독은 `subscriptionOfferDetails`의 `offerToken` 필요, 평생 상품은 불필요 — 상품 타입별 분기
  - `queryPurchases()`: `queryPurchasesAsync`로 SUBS/INAPP 각각 조회 → 활성 구매 있으면 `isPremium=true` + `SettingsRepository.updatePremium()` 동기화 → 위젯/연속 기록 보호 로직에서 참조
  - `onPurchasesUpdated`: 구매 콜백에서 `acknowledgePurchase` 처리 후 `queryPurchases()` 재조회로 상태 갱신
  - 복원 버튼은 별도 API 없이 `queryPurchases()` 재호출로 충분 (Play는 계정에 귀속되어 자동 동기화)
- **PremiumScreen**: 기능 5종 카드 + 플랜 선택 카드 3종(월/연/평생) + 구매/복원 버튼, 이미 구독 중이면 안내 화면으로 분기
- **연속 기록 보호**: `WaterRepository.updateStreak(record, current, isPremium)` — `lastAchievedDateKey`가 정확히 2일 전이고(하루만 놓침) 프리미엄이며 이번 달 보호를 아직 안 썼을 때만 스트릭을 끊지 않고 유지. 2일 이상 공백은 보호 대상 아님(기존 로직대로 1로 리셋)
